# How to capture screenshots / demo GIF

The README references three images under `docs/screenshots/`:

| File | What it should show | How |
|---|---|---|
| `appicon.png` | Project hero / logo | Auto-generated by `python tools/make-icon.py`. Don't edit by hand. |
| `mainwindow-light.png` | The main window in Light theme with a real queue mid-copy (progress bar visible, a Done item, an InProgress item, a Queued item with the ✕ Remove button) | Capture manually — see below. |
| `mainwindow-dark.png` | Same, in Dark theme. Useful because the custom title bar + InfoBar look noticeably different. | Capture manually — see below. |
| `demo.gif` *(optional)* | Short clip: dragging a folder, hitting Start, removing an item mid-copy, completion toast. ~6–10 s. | [ScreenToGif](https://www.screentogif.com/) is the easiest tool on Windows. |

## Capturing the still PNGs

1. Build & launch the unpackaged exe:
   ```pwsh
   dotnet build RoboCopyGUI.csproj -p:Platform=x64
   .\bin\x64\Debug\net8.0-windows10.0.19041.0\win-x64\RoboCopyGUI.exe
   ```
2. Set up a representative queue: drop a folder with ~50 files (~1 GB total
   is a good demo) so the progress bar has meaningful motion.
3. Hit **Start**. Wait until at least one item is Done, one is InProgress,
   and one is still Queued (so the ✕ Remove affordance is visible in the
   shot).
4. Use **Win+Shift+S** (Windows Snipping Tool) → Window mode → click the
   RoboCopyGUI window. Save as `mainwindow-<theme>.png` here in
   `docs/screenshots/`.
5. Toggle the theme via the sun/moon button (or **Ctrl+T** if you wire one
   up) and repeat for the other theme.

## Capturing the demo GIF (optional)

1. Install ScreenToGif (free, open source).
2. Record at 15 fps, target window only, 600px wide max.
3. Trim, then export as GIF (Lossy, 192-color palette) so the file lands
   under ~3 MB — GitHub starts complaining at 5 MB and the README will
   look bad if it has to scroll.
4. Save as `docs/screenshots/demo.gif`.

## Replacing the README placeholders

The README uses HTML `<img>` tags (not Markdown) so we can set a max width
to avoid them dominating the page. After you drop the PNGs in, no
README edit is needed — the existing references will start rendering.
